# Incident Handler's Journal - Cybersecurity Portfolio

## Overview
A professional incident handler's journal template designed for documenting cybersecurity incidents, investigations, and tool analyses. This template provides a structured format for security professionals to record their findings, track investigations, and maintain comprehensive incident response documentation aligned with industry standards.

## Features
- Structured journal entry format for consistent documentation
- Integration of the 5 W's methodology for thorough incident analysis
- Tool usage documentation section
- NIST Incident Response Lifecycle phase identification
- Reflection and notes section for continuous improvement
- Multiple entry templates for extensive incident tracking

## Template Structure
Each journal entry includes the following sections:
1. **Date**: Record the date of the journal entry
2. **Entry**: Record the journal entry number
3. **Description**: Brief description of the journal entry (20-50 words)
4. **Tool(s) Used**: List of cybersecurity tools utilized during the activity
5. **The 5 W's**: Complete incident documentation covering:
   - Who caused the incident
   - What happened
   - When the incident occurred
   - Where the incident happened
   - Why the incident happened
6. **Additional Notes**: Space for supplementary thoughts, questions, or findings

## Minimum Requirements
To create a comprehensive journal, include at minimum:
- 4 dated and numbered journal entries
- 2 entries documenting incident investigations using the 5 W's methodology
- 2 entries describing cybersecurity tool usage
- Reflection section addressing learning journey and challenges

## Supported Activities
This journal template supports documentation for the following cybersecurity activities:
- Documenting incidents with an incident handler's journal
- Analyzing network packets
- Capturing network packets
- Investigating suspicious file hashes
- Using playbooks to respond to attacks
- Reviewing final reports
- Exploring signatures and logs with Suricata
- Performing queries with Splunk
- Performing queries with Chronicle

## NIST Incident Response Lifecycle Integration
Identify which phase(s) of the NIST Incident Response Lifecycle each investigation occurred in:
1. Preparation
2. Detection and Analysis
3. Containment, Eradication, and Recovery
4. Post-Incident Activity

## How to Use
1. Download the template file
2. Create entries for each cybersecurity activity or investigation
3. Complete all sections for each journal entry
4. Review entries for completeness, accuracy, and proper grammar
5. Update entries to include NIST lifecycle phase identification where applicable
6. Complete reflection questions in the designated section

## Reflection Questions
Address the following questions in the Reflections/Notes section:
1. Were there any specific activities that were challenging? Why or why not?
2. Has your understanding of incident detection and response changed since taking this course?
3. Was there a specific tool or concept that you enjoyed most? Why?

## Quality Assurance Checklist
Before submission, ensure your journal contains:
- 4 completed journal entries with Date, Entry, and Description sections filled
- 2 entries with complete 5 W's incident investigation documentation
- 2 entries with detailed cybersecurity tool usage descriptions
- Proper grammar, punctuation, and spelling throughout
- Reflections section addressing all three reflection questions

This journal serves as:
- A learning tool for cybersecurity training
- A portfolio piece demonstrating incident response skills
- A reference document for future investigations
- Evidence of structured analytical thinking and documentation skills
